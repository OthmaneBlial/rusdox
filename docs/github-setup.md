# GitHub Setup

This page covers the repository settings that still need GitHub web UI clicks after you push.

RusDox already includes:

- README
- docs
- LICENSE
- CONTRIBUTING guide
- CODE_OF_CONDUCT
- SECURITY policy
- SUPPORT guide
- issue forms
- PR template
- CI workflow
- release workflow
- social preview asset file

## Settings To Configure On GitHub

### 1. About section

Set a short description close to:

`Generate DOCX and PDF from YAML at Rust speed.`

### 2. Topics

Suggested topics:

- rust
- yaml
- docx
- pdf
- ooxml
- document-generation
- cli
- templates

### 3. Social preview image

Upload:

- [../assets/social-preview-rusdox.png](../assets/social-preview-rusdox.png)

### 4. Discussions

Enable GitHub Discussions so usage questions do not become noisy bug issues.

### 5. Community profile

After pushing these files, check the GitHub community profile and complete any remaining recommendations.

### 6. Profile and pinning

Pin RusDox on your GitHub profile and mention it in your profile README.

## Maintainer Tip

When you add a new example or big feature:

1. regenerate examples
2. regenerate gallery assets
3. update docs if authoring changes
4. update the social preview if positioning changes
